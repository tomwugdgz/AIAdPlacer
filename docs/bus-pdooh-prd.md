## 一、产品目标

### 1.1 背景

XX科技已构建 AI 驱动的 pDOOH（程序化数字户外广告）投放平台 AIAdPlacer，当前覆盖社区门禁媒体资源。公交车身广告作为城市主流户外媒体，具备**覆盖广、触达高频、线路固定可预测**的特性，是 pDOOH 生态的关键补充。

### 1.2 产品目标

| 目标 | 说明 | 衡量指标 |
|------|------|----------|
| **G1：程序化接入公交媒体资源** | 将公交车身广告资源标准化、程序化，接入 AIAdPlacer 竞价引擎 | 接入城市数 ≥ 10，线路数 ≥ 50 |
| **G2：智能线路推荐** | 基于腾讯地图 API + 移动人口数据，对线路进行热力评分，自动推荐高价值线路 | 推荐线路 CTR 提升 ≥ 20% |
| **G3：多维度竞价引擎** | 支持覆盖人群、线路热点、线路等级、时段溢价等多维竞价 | 竞价响应时间 < 500ms |
| **G4：AI 方案审核与归因** | 用 LLM 对历史投放方案进行智能审核，投放后自动效果归因 | 方案审核时间 < 30s，归因报告自动生成 |

### 1.3 产品范围

**纳入范围（Scope In）**：
- 公交线路资源管理（录入、分级、定价）
- 线路热力评分（腾讯地图 API + 人口数据）
- 多维度竞价引擎（人群/热点/等级/时段）
- AI 历史方案审核（LLM 评审）
- 投放效果归因（展示量、触达人数、转化追踪）

**暂不纳入（Scope Out）**：
- 公交车内广告（把手、座椅、车顶）
- 实时 GPS 轨迹追踪（二期）
- 跨城公交线路联动（二期）

## 三、需求池

### P0（核心功能，MVP 必须完成）

| ID | 需求 | 说明 | 验收标准 |
|----|------|------|----------|
| REQ-01 | 线路资源录入与管理 | 支持 Excel 批量导入线路（城市、线路名、级别、车辆数、单价） | 成功导入 ≥ 95 条线路，数据准确率 100% |
| REQ-02 | 线路热力评分 | 调用腾讯地图 API 获取线路沿途 POI + 移动人口热力数据，输出 0-100 评分 | 评分计算 < 2s，与人工评估相关性 ≥ 0.7 |
| REQ-03 | 线路等级体系 | 支持 S / A++ / A+ / A 四级分类，每级对应基准单价 | 等级与单价联动，修改等级自动更新报价 |
| REQ-04 | 基础竞价引擎 | 根据覆盖人群、线路等级、投放时长计算竞价价格 | 价格计算 < 500ms，公式可配置 |
| REQ-05 | 线路搜索与筛选 | 按城市、等级、价格区间、热力评分筛选线路 | 筛选结果 < 1s 返回 |
| REQ-06 | 投放方案创建 | 选择线路 + 时段 + 预算，生成投放方案草稿 | 方案可保存、编辑、提交审核 |

### P1（重要功能，V1.1 完成）

| ID | 需求 | 说明 | 验收标准 |
|----|------|------|----------|
| REQ-07 | AI 方案审核 | LLM 评审方案合规性（品牌与线路匹配度、预算合理性） | 审核 < 30s，准确率 ≥ 85% |
| REQ-08 | 时段溢价规则 | 支持高峰时段（早晚通勤）溢价系数配置 | 溢价计算自动生效 |
| REQ-09 | 历史方案库 | 存储历史投放方案，支持相似方案检索 | 检索 < 2s，Top5 结果相关 |
| REQ-10 | 效果归因报告 | 投放后自动生成报告（展示量、触达人数、成本效率） | 报告可导出 PDF |
| REQ-11 | 多方案对比 | 并排对比多套方案的线路、价格、预估触达 | 对比视图清晰直观 |

### P2（增强功能，后续版本）

| ID | 需求 | 说明 |
|----|------|------|
| REQ-12 | 实时 GPS 轨迹接入 | 接入公交实时位置，动态优化投放 |
| REQ-13 | 跨城线路联动包 | 支持多城市线路打包投放 |
| REQ-14 | 创意素材 DCO | 根据线路人群特征，动态优化广告素材 |
| REQ-15 | 竞争对手屏蔽 | 同一线路避免竞品同时投放 |

## 四、数据模型

### 4.1 线路资源（BusRoute）

| 字段 | 类型 | 说明 |
|------|------|------|
| route_id | UUID | 唯一标识 |
| city | String | 城市（深圳/广州/北京...） |
| route_name | String | 线路名称（如"深圳三侧"） |
| route_code | String | 线路编号 |
| level | Enum | S / A++ / A+ / A |
| vehicle_count | Integer | 车辆数 |
| monthly_price | Decimal | 月均单车价格（元） |
| coverage_population | Integer | 日均覆盖人群（腾讯地图 API） |
| heat_score | Float | 热力评分（0-100） |
| pois | JSON | 沿途 POI 列表 |
| status | Enum | 可预订 / 已预订 / 下线 |
| created_at | DateTime | 创建时间 |

### 4.2 投放方案（BusCampaign）

| 字段 | 类型 | 说明 |
|------|------|------|
| campaign_id | UUID | 唯一标识 |
| advertiser_id | UUID | 广告主 ID |
| route_ids | UUID[] | 选定线路 |
| start_date | Date | 开始日期 |
| end_date | Date | 结束日期 |
| total_budget | Decimal | 总预算 |
| ai_review_status | Enum | 待审核 / 通过 / 驳回 |
| ai_review_comment | Text | AI 审核意见 |
| attribution_report | JSON | 效果归因数据 |
