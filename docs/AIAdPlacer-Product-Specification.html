1. 产品概述

AI Native pDOOH 平台，LLM+Agent+RAG+MCP 全栈落地，5V数据模型（V4门禁独家），目标用户：广告主/媒体主/代理商/AI Agent。

📄 查看完整 Markdown 版本

2. 核心业务概念

pDOOH vs 传统户外、5V数据模型（V1人口/V2消费/V3社区/V4门禁/V5行为）、OTC模型（PV×Reach×Frequency×系数）、社区广告资源类型（单元门/道闸/App/LED）。

3. 系统架构

整体四层架构：用户交互层 → FastAPI后端（v2.0.0 CPS）→ LangGraph Agent编排层（4个专业Agent+统一编排）→ 数据与服务层（PostgreSQL/Redis/ChromaDB/腾讯地图/5V数据）。目录结构完整说明。

4. 功能模块详解

五大模块：①人群洞察（KMeans聚类，POST /api/v2/agents/audience-insight）；②智能排期（FM+deepMCP，预算约束优化）；③动态创意（AIGC+DCO，多模态组合）；④效果归因（跨端匹配，V4门禁验证）；⑤RAG知识库（ChromaDB+text2vec，行业报告/历史案例检索）。

5. 用户使用流程

广告主8步流程：注册→创建计划→AI推荐点位→AI生成排期→AI生成创意→确认支付→投放执行→效果报告。代理商可管理多广告主、AI批量生成报告、监控竞品。

6. AI Agent 编排逻辑

统一编排Agent（Orchestrator）三阶段：规划（分析请求决定调用哪些Agent）→执行（并行/串行）→反思（检查结果，不满意最多重试3轮）。Agent间用结构化JSON+JSON Schema通信。

7. 数据模型

四张核心表：users（用户）/campaigns（投放计划）/creatives（创意素材）/otc_logs（OTC归因日志）。5V数据层通过统一DataProvider接口访问，支持query()和get_coverage()方法。

8. API 完整规范

认证：X-API-Key Header。Agent调用：POST /api/v2/agents/{agent_name}/invoke（通用输入输出Schema）。RAG查询：POST /api/v2/rag/query（top-k相似检索）。MCP工具：POST /api/v2/mcp/tools/{list,invoke}。

9. MCP/A2A 集成

MCP：提供search_community / estimate_otc / generate_creative / query_campaign / get_attribution_report 五个工具，FastAPI实现，支持SSE流式返回。A2A：暴露Agent Card（/.well-known/agent.json），支持Agent发现、异步任务、流式进度。

10. 部署与运维

部署架构：Nginx反向代理→FastAPI多worker→PostgreSQL主从+Redis哨兵+ChromaDB持久化。环境变量完整配置说明。监控：Prometheus+Grafana，ELK日志，告警规则（错误率>5%/执行>30s/连接池>80%）。

11. 行业标准对齐

T/CCSA 738-2025（中国通信标准化协会程序化户外广告标准）。国际对标：IAB OpenRTB 3.0、OMA标准、GDPR、PIPL。OTC模型符合「可见曝光」定义。

12. 产品评价：喜欢与改进空间

✅ 喜欢：AI Native设计、5V数据壁垒、MCP/A2A前瞻、OTC模型实用。

⚠️ 改进空间：①数据隐私合规（联邦学习/聚合统计/opt-in）；②Agent可靠性（人在回路/置信度评分/决策日志）；③创意质量（质量评估模型/A/B测试/品牌素材上传）；④跨屏归因（概率归因/因果推断）；⑤性能成本（LLM缓存/大小模型分工/批量处理）；⑥用户体验（React前端/拖拽编辑/实时大屏/移动App）；⑦生态建设（媒体主API/pDOOH联盟链/电商全链路追踪）。
